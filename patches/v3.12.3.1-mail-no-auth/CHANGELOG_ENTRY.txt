3.12.3.1 - 2026-04-27: hot-fix no-auth SMTP relay 寄信失敗 - 用戶套 v3.12.3.0 後巡檢實跑寄信報「smtp auth extension not supported by server」, 根因: scripts/generate_report.py 自己有一套 SMTP 邏輯, 原 if cfg.get("smtp_user"): srv.login(user, pass) 只檢 smtp_user 沒檢 smtp_pass, 對「smtp_user 有值 (e.g. alien.lee@cathaysec.com.tw) 但 smtp_pass 空 (relay 不需 auth)」場景, 會硬呼叫 srv.login(user, "") → relay server 不 advertise AUTH 直接 reject. 修法 (對齊 services/email_service.py 邏輯): user AND pass 都非空才 login, 同時支援 smtp_pass 為 ENV:VAR_NAME 前綴 (從環境變數解). 用戶眼下不需做別的, 直接套 patch + 下次巡檢就會通. 純 generate_report.py 改 1 個 if 區塊, 其他不動. 已知歷史殘留: generate_report.py 與 services/email_service.py 仍是平行兩套 SMTP 邏輯, 此 patch 只把 login 行為對齊, 完整統一 refactor 留待未來.
