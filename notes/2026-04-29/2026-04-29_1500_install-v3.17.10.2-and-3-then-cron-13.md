## Step 4. 手動觸發第一次採集

點 toolbar 的「📡 採集」按鈕 → 變「⏳ 採集中...」。
等 1-3 分鐘 (背景跑 ansible-playbook collect_connections.yml)。

完成後會跳 alert：
- `✓ 採集完成 新增邊: N / 更新: N` → 成功, 跳 Step 5
- `✗ 採集失敗: ...` → 跳 Step 6 診斷

---

## Step 5. 採集成功 → 看線出來 + 設 cron

### 5a. 重整看線
拓撲應從 0 邊變有邊, 自動切回 hierarchical layout (LR 樹狀 + 箭頭)。

### 5b. 設 cron 自動排程
進 `/admin` → 「拓撲管理」tab → 找「⏰ 採集排程 (cron)」section：
- interval: **30 分** (推薦. 5/10 太密, 60 太疏)
- ☑ **上班時間限定** (9-18 週一~五)
- limit_hosts: 留空 (= 採全部 4 台)

按「儲存」。預期會在 sysinfra 的 crontab 寫一行：
```
*/30 9-18 * * 1-5 /opt/inspection/scripts/run_dep_collect.sh # MARK_DEP_COLLECT
```

### 5c. 確認 cron 真的寫進去
```bash
sudo crontab -u sysinfra -l | grep MARK_DEP_COLLECT
```

收工。之後 cron 自動跑, 看時點「📊 狀態」確認最近一次 OK 即可。

## Step 6. 採集失敗診斷

### 6a. 點「📊 狀態」看 error 欄

UI alert 會顯示：
```
status:        failed
error:         <關鍵訊息>
```

### 6b. 看 ansible log 細節

```bash
sudo tail -80 $(sudo ls -t /opt/inspection/logs/dep_collect_*.log 2>/dev/null | head -1)
```

### 6c. 對照常見錯誤

| error 訊息 | 原因 | 修法 |
|---|---|---|
| `unreachable` / `No route to host` | 13 無法 SSH 到 host (網路 / hosts.yml IP 錯) | 改 `ansible/inventory/hosts.yml` 確認 IP, `ansible all -m ping` 試連 |
| `Permission denied (publickey)` | sysinfra SSH key 沒分發到 host | `sudo -u sysinfra ssh-copy-id sysinfra@<host>` |
| `sudo: a password is required` | 該 host 沒設 NOPASSWD | 在 host 上建 `/etc/sudoers.d/sysinfra-nopasswd` (memory 規則, 別碰 ansible_svc) |
| `TASK [...] FAILED` 帶 ss command | 該 host 沒裝 iproute2 (`ss` 缺) | `dnf install iproute -y` |

### 6d. 修完重試

回到 Step 4 點「📡 採集」再試。確認成功才接 Step 5 設 cron。
