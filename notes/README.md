# notes/ — 巡檢系統診斷筆記本

Claude 每次要你執行測試 / 檢查時，會寫一份新檔放在此目錄。
公司端可直接在 GitHub 網頁打開看，不需 mail 來回。

## 目錄結構

**每日一個子目錄**，避免根目錄單日太多檔：

```
notes/
├── README.md
├── 2026-04-23/          ← 4/23 當天所有 notes
│   ├── 2026-04-23_0812_twgcb-empty-data.md
│   ├── 2026-04-23_1600_TODO-NOW.md
│   └── ...
└── 2026-04-24/          ← 下一天
    └── ...
```

## 命名規則

檔案本身仍用完整日期時間開頭：`YYYY-MM-DD_HHMM_主題.md`

方便搜尋 + 即使檔案被搬出資料夾也能看出日期。

時間以 Claude 本機（台北時間）為準。

## 使用流程

1. Claude 寫新筆記本 → commit & push
2. 你在公司打開 GitHub：https://github.com/alienid4/it-web-full/tree/main/notes
3. 按筆記指示執行（通常是打 URL / 跑指令 / 看設定檔）
4. 把結果貼回聊天，Claude 接著判讀

## 筆記本結構

每份筆記本都有固定區塊：

- **背景** — 此次要查什麼、為何
- **要執行的測試** — URL / 指令 / 操作步驟
- **預期結果** — 什麼是正常、什麼是異常
- **判讀表** — 可能情境 → 下一步
- **回傳格式** — 你該把什麼貼回聊天
