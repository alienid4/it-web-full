# 2026-04-23 13:20 — v3.11.13.0 帳號盤點主機搜尋 (Tarball patch)

## 新功能

帳號盤點頁 `/audit` 在主機下拉右邊新增「主機搜尋」input，placeholder `🔍 hostname / IP / OS`。

- 與原主機下拉可**同時使用**（AND 關係）
- 搜尋目標：`hostname` + `ip` + `os` + `system_name`（fetch `/api/hosts` 建 map 後比對）
- 即時過濾（oninput 觸發 `renderAuditTable()`）
- 完全前端實作，不影響 API

## 檔案改動

單檔：`webapp/templates/audit.html`

HTML 新增一行 input；JS 新增 `_hostMap` 全域變數 + DOMContentLoaded 多 fetch `/api/hosts` + `renderAuditTable` 加 hostKw 比對邏輯。

## 套用（13 上，走傳統 tarball 流程）

### Step 1 — 下載 tarball

在公司 Win10 下載：

```
https://github.com/alienid4/it-web-full/releases/download/v3.11.13.0/patch_v3.11.13.0.tar.gz
```

scp / USB 搬到 13 的 `/tmp/`。

### Step 2 — 解壓 + 跑 patch_apply.sh

```bash
cd /tmp
tar xzf patch_v3.11.13.0.tar.gz
sudo ./v3.11.13.0/patch_apply.sh
```

會互動式問「確定套用此 patch? (y/n)」，按 `y`。

### Step 3 — 驗證

打開 `http://10.92.198.13:5000/audit`，應看到：

- **主機** 下拉後面多了「**主機搜尋**」input
- placeholder `🔍 hostname / IP / OS`
- 輸入 `198.13` → 只剩 13 的帳號
- 輸入 `rhel` → 只剩 RHEL 主機的帳號
- 清空 input → 恢復全部

### Step 4 — Rollback（如有問題）

patch_apply.sh 會自動備份到 `/tmp/itagent_pre_patch_3.11.13.0_*.tar.gz`，rollback 指令：

```bash
sudo systemctl stop itagent-web
sudo tar xzf /tmp/itagent_pre_patch_3.11.13.0_*.tar.gz -C /opt/
sudo systemctl start itagent-web
```
