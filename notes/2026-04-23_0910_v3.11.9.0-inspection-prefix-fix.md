# 2026-04-23 09:10 — v3.11.9.0 治本：inspection JSON 加前綴

## 治本動機

reports/ 底下各種 JSON 都有**明確前綴**，唯獨 inspection 沒有：

| 前綴 | 來源 |
|---|---|
| `twgcb_` | check_twgcb role |
| `packages_` | collect_packages role |
| `network_` | check_network role |
| `nmon_` | collect_nmon role |
| ~~無前綴~~ 只用 timestamp | site.yml inspection ← **異類** |

→ seed_data.py 想 glob inspection 必須用 regex 識別 timestamp 格式，容易誤判。

## 本次改動

### 1. `site.yml` 產出加 `inspection_` 前綴

```yaml
# Before
dest: "{{ inspection_home }}/data/reports/{{ report_timestamp }}_{{ inventory_hostname }}.json"

# After
dest: "{{ inspection_home }}/data/reports/inspection_{{ report_timestamp }}_{{ inventory_hostname }}.json"
```

傳給 `generate_report.py` 的 prefix 也同步加 `inspection_`，HTML 報告產生不受影響。

### 2. `seed_data.py` 兼容兩種格式

```python
TS_RE = re.compile(r"^(?:inspection_)?\d{8}_\d{6}_")
```

**同時接受**：
- `inspection_20260423_084220_SECSVR198-013T.json`（v3.11.9.0+ 新格式）
- `20260423_084220_SECSVR198-013T.json`（舊格式，向後相容 — 現存 reports/ 裡的檔都能繼續讀）
