# 2026-04-23 14:36 — v3.11.16.0 高權限 Phase 2 人工標記 + 強制備註

## 新功能

### UI

- 表格多一個**操作**欄
  - 未標記 → 橘色「**⭐ 標記**」按鈕（點開 modal）
  - 人工已標記 → 灰色「**取消⭐**」按鈕（confirm 後 DELETE）
  - UID=0/GID=0 系統高權限 → 顯示「系統」文字（不能人工改）
- 點「⭐ 標記」跳 modal：
  - **備註（必填）**：為什麼是高權限（例：AP 管理員、DBA）
  - 標記人（選填）：例：IT 主管 / AP 負責人
  - 空備註會擋
- 兩種 badge：
  - `🔴 系統高權限`（UID=0 / GID=0）
  - `⭐ 人工高權限`（hover 顯示 reason + marked_by tooltip）

### 後端

新 3 個 endpoint（全部 `@login_required`）：
- `POST /api/audit/privileged` — 強制 reason 非空否則 400
- `DELETE /api/audit/privileged`
- `GET /api/audit/privileged` — Phase 3 簽核頁會用

新 collection `account_privileged_flags`：
```
{hostname, user, reason, marked_by, marked_at}
unique index: (hostname, user)
```

順便修 **v3.11.15.0 漏傳 bug**：`_get_audit_data` 的 result 沒把 gid 傳給前端，這次補上。

## 套用（2 檔單檔 cp，建議路線）

`patch_apply.sh` 的備份問題還沒解（debug 待你回傳），**建議直接 2 檔 cp 走**。

### Step 1 — 下載 tarball

```
https://github.com/alienid4/it-web-full/releases/download/v3.11.16.0/patch_v3.11.16.0.tar.gz
```

### Step 2 — 解壓 + 單檔覆蓋 + restart

```bash
cd /tmp
curl -LO https://github.com/alienid4/it-web-full/releases/download/v3.11.16.0/patch_v3.11.16.0.tar.gz
tar xzf patch_v3.11.16.0.tar.gz

TS=$(date +%Y%m%d_%H%M)

# audit.html
sudo cp /opt/inspection/webapp/templates/audit.html \
        /opt/inspection/webapp/templates/audit.html.bak.${TS}
sudo cp /tmp/v3.11.16.0/files/webapp/templates/audit.html \
        /opt/inspection/webapp/templates/audit.html
sudo chown sysinfra:itagent /opt/inspection/webapp/templates/audit.html

# api_audit.py
sudo cp /opt/inspection/webapp/routes/api_audit.py \
        /opt/inspection/webapp/routes/api_audit.py.bak.${TS}
sudo cp /tmp/v3.11.16.0/files/webapp/routes/api_audit.py \
        /opt/inspection/webapp/routes/api_audit.py
sudo chown sysinfra:itagent /opt/inspection/webapp/routes/api_audit.py

sudo systemctl restart itagent-web
```

### Step 3 — 驗證

打開 `/audit`：

1. 表格最右多「操作」欄
2. root（UID=0）顯示「系統」灰文字
3. 其他帳號（例如 sysinfra、alien）顯示橘色「⭐ 標記」按鈕
4. 點「⭐ 標記」跳 modal → 填備註 → 確定
5. 該帳號變紅底 + `⭐ 人工高權限` badge + 操作欄變灰色「取消⭐」
6. Hover `⭐ 人工高權限` badge 看 tooltip (reason + marked_by)
7. 點「取消⭐」→ confirm → 恢復

**不需重跑 run_inspection.sh**（這次只改前後端，ansible role 沒動）。
