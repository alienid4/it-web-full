# 2026-04-23 16:17 — v3.11.20.0 方案 X：onboard 腳本自動 insert DB

## 需求（你確認的）

> 加新主機後系統自動跑 Dashboard / 今日報告 / 帳號盤點 / TWGCB / 軟體盤點 / 效能月報 6 個功能。
> 一條指令搞定不要先 UI 填表。

## 改動

擴充 `onboard_new_host.sh` 參數 + 新增 Step 0：

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh <hostname> [ip] [os_group] [ssh_user]
```

- `hostname` **必填**
- `ip` 新主機不在 DB 時**必填**（已在 DB 可省）
- `os_group` 選填，預設 `linux`
- `ssh_user` 選填，預設 `sysinfra`

**Step 0**：若 DB 沒這筆，自動 `mongosh insertOne` 帶最少欄位：
```js
{
  hostname, ip, os_group, ssh_user, ssh_port:22,
  ssh_key: '/home/sysinfra/.ssh/id_ed25519',
  connection: 'ssh',
  nmon_enabled: false,
  tier: '', department: '', ap_owner: '', custodian: '', system_name: '',
  created_at, onboarded_by: 'onboard_new_host.sh'
}
```

業務欄位（tier/部門/ap_owner/custodian）事後用 UI `/admin#hosts` 點該主機補。

## 使用範例

### 情境 A：加第 3 台 linux（全新，DB 沒）

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh SECSVR198-012T 10.92.198.12
```

腳本自動：
1. [0/6] insert 到 DB
2. [1/6] 驗證 DB
3. [2/6] 重建 inventory
4. [3/6] ansible ping
5. [4/6] run_inspection.sh（完整巡檢 + packages + seed + CIO）
6. [4.5] TWGCB 掃描
7. [5/6] 啟用 nmon
8. [6/6] 印 6 個驗證 URL

約 3~7 分鐘結束。

### 情境 B：加主機自訂 ssh_user

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh \
  SECSVR198-012T 10.92.198.12 linux ansible_svc
```

### 情境 C：主機已在 DB（之前 UI 填過）

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh SECSVR198-012T
```

Step 0 檢查 DB 有就 skip insert，繼續跑 1~6。

## 要加第 3 台時

```bash
# 一條指令搞定 (約 3~7 分鐘)
sudo /opt/inspection/scripts/onboard_new_host.sh <新hostname> <新IP>

# 範例
```

跑完後 F5 這 6 個頁面應該都有第 3 台：

| 頁 | URL | 看什麼 |

| Dashboard | `/` | 第 3 張主機卡 + CPU/MEM/Disk/Swap |
