# 2026-04-23 14:59 — v3.11.17.0 關鍵修復：seed 漏匯 account_audit

## 根因

從你 mongosh 回「找不到 root」確認：

`account_audit` collection 完全是空的 — 即使你跑過 `run_inspection.sh`、ansible role 也採集了、inspection_*.json 裡明明有 `results.account_audit` 陣列。

原因：**seed_data.py 只把 inspection_*.json 匯到 `inspections` collection，忽略 `results.account_audit`**。而 api_audit.py 讀的是獨立 `account_audit` collection → 永遠空。

這是 v3.4.7.0 改 api_audit 讀新 collection 時埋的 bug，一直沒爆到今天。

## 修法（已 commit）

`seed_data.py` `import_inspections()` 新增 loop：

```python
for a in results.account_audit:
    db.account_audit.update_one(
        {"hostname": hostname, "user": a.user, "run_date": run_date},
        {"$set": {...a, "hostname": hostname, "run_date": run_date}},
        upsert=True,
    )
```

每跑一次 seed_data.py，會把 `reports/inspection_*.json` 下**所有歷史 account_audit** 全部灌進 collection。

## 套用（13 上，1 檔 cp + 重跑 seed）

### Step 1 — 下載單檔

```
https://github.com/alienid4/it-web-full/raw/main/AI/webapp/seed_data.py
```

scp 到 13 的 `/tmp/seed_data.py`。

### Step 2 — 覆蓋 + 重跑 seed（不用重跑 ansible，直接讀現有 JSON）

```bash
TS=$(date +%Y%m%d_%H%M)

sudo cp /opt/inspection/webapp/seed_data.py \
        /opt/inspection/webapp/seed_data.py.bak.${TS}
sudo cp /tmp/seed_data.py /opt/inspection/webapp/seed_data.py
sudo chown sysinfra:itagent /opt/inspection/webapp/seed_data.py

# 重跑 seed (不用重跑 run_inspection.sh, JSON 已存在)
cd /opt/inspection/webapp && sudo -u sysinfra python3 seed_data.py
```

**預期輸出**：
```
hosts: 2 筆匯入/更新
inspections: N 筆匯入/更新
account_audit: M 筆 (累計)    ← ✅ 這行是新的
settings: 匯入完成
```

`M` 應該是 兩台主機 × 各 N 個帳號 = 一堆（幾十筆起跳）。

### Step 3 — 驗證

```bash
mongosh inspection --quiet --eval '
  const r = db.account_audit.findOne({user:"root"});
  if (!r) print("❌ 還是找不到 root");
  else {
    print("✅ root: hostname=" + r.hostname + " uid=" + r.uid + " gid=" + r.gid);
  }
  print("總筆數:", db.account_audit.countDocuments({}));
'
```

**預期**：`✅ root: ... uid=0 gid=0` + 總筆數幾十。

### Step 4 — 刷 /audit

重整 `http://10.92.198.13:5000/audit`，應該：
- 「最後盤點: 2026-04-23」（不是「尚未執行」）
- 表格有滿滿的帳號
- root 紅底 + `🔴 系統高權限` badge
- 其他帳號有 `⭐ 標記` 按鈕
- 最底下 `/etc/passwd 原文檢視` 正常運作

不需要重啟 Flask，也不需要重跑 run_inspection.sh。
