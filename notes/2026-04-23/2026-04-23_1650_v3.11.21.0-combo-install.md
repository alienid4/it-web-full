# 2026-04-23 16:50 — v3.11.21.0 combo pack 一鍵裝

## 為什麼出這版

前面用 `curl raw URL` 下載 `onboard_new_host.sh` 覆蓋，但你看到的 Usage 還是 v3.11.18 舊版（`[4/5]` 不是 `[4/6]`）。
可能原因 (raw URL cache / proxy 轉 HTML / cp 沒生效)，**這版改走 GitHub Releases CDN**（已證實公司網路可通），tarball 內建 `install.sh` 自動備份+覆蓋+驗證。

## 套用（一條指令全搞定）

13 上執行：

```bash
cd /tmp
rm -rf v3.11.21.0 patch_v3.11.21.0.tar.gz
curl -fLO https://github.com/alienid4/it-web-full/releases/download/v3.11.21.0/patch_v3.11.21.0.tar.gz
tar xzf patch_v3.11.21.0.tar.gz
sudo ./v3.11.21.0/install.sh
```

install.sh 會自動：

1. 驗證 tarball 內 onboard_new_host.sh 含「方案 X」字樣（來源不對就 abort，不會亂覆蓋）
2. 備份舊版到 `.bak.<時間戳>`
3. cp + chown + chmod
4. 驗證目的地確實是新版
5. 更新 version.json（右下角變 3.11.21.0）
6. `systemctl restart itagent-web`
7. HTTP ping 確認 Flask 起來
8. 印驗證指令 + rollback 指令

## 之後加第 3 台主機

一條：

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh <新hostname> <新IP>
```

範例：

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh SECSVR198-012T 10.92.198.12
```

約 3~7 分鐘，6 個頁面自動有第 3 台。

---

## Rollback（若出錯）

install.sh 結尾會印具體 rollback 指令，大概長這樣：

```bash
sudo cp /opt/inspection/scripts/onboard_new_host.sh.bak.<TS> /opt/inspection/scripts/onboard_new_host.sh
sudo cp /opt/inspection/data/version.json.bak.<TS> /opt/inspection/data/version.json
sudo systemctl restart itagent-web
```
