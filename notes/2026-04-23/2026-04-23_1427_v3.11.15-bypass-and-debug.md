## 順便 debug：patch_apply.sh 為何卡備份

在 13 上跑這條（**不丟 stderr**）：

```bash
cd /tmp
sudo tar czf /tmp/debug_backup_test.tar.gz \
  --exclude=inspection/container \
  --exclude=inspection/logs \
  -C /opt inspection
echo "---rc=$?---"
ls -la /tmp/debug_backup_test.tar.gz 2>&1
```

**貼完整輸出給我**（即使失敗的訊息也貼）。可能看到：

| 訊息特徵 | 根因 | 我下版會怎麼改 |
|---|---|---|
| `No space left` | /tmp 空間不夠 | 改把 BACKUP_FILE 路徑改 `/var/backups/` |
| `Permission denied`: `.vault_pass` 或 `config.py` | tar 讀 600 權限檔卡 root 外 UID | patch_apply.sh 加 `--warning=no-file-ignored` 或先 chmod |
| `file changed as we read it` | Flask 寫 log 時被 tar 讀 | 備份前先 `systemctl stop itagent-web` |
| `Cannot stat`: 某 socket / pipe | container/mongodb socket 外漏 | exclude 加 `--exclude=inspection/data/uploads/*.sock` 之類 |
| `cannot open directory ... container/mongodb_data`: Permission denied | podman UID 999 的 dir root 也進不去（unlikely 但可能）| exclude 加整個 `data/` 或跑 `sudo` 之前切 root |
