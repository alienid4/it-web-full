# 2026-04-23 15:58 — v3.11.19.0 治本修 patch_apply.sh 備份失敗

## 為什麼之前一直卡

`patch_apply.sh` 原本的備份段：
```bash
tar czf ... 2>/dev/null || fail "備份失敗"
```

`2>/dev/null` **吞掉所有 tar 錯誤訊息**，使用者只看到「備份失敗」但不知道原因。而且 tar 沒加任何 `--ignore-failed-read` 之類的寬容參數，遇到任一檔無法讀就整個 fail。

這次一起治本。

---

## 改動

| 項目 | Before | After |
|---|---|---|
| stderr 處理 | `2>/dev/null` 全吞 | 存 `/tmp/itagent_backup_err_$$.log`, 失敗時印 |
| tar 寬容參數 | 無 | `--ignore-failed-read`、`--warning=no-file-{ignored,changed,removed}` |
| exclude | `container/`, `logs/` | + `*.sock`, `*.pid`, `*.swp`, `*.pyc`, `__pycache__` |
| 備份路徑 | 固定 `/tmp/` | 優先 `/var/backups/inspection/`, 無寫權限 fallback `/tmp` |
| 失敗處置 | 直接 exit | 印原因清單 + 問「無備份繼續?」 y/n |

## 套用（一次性，以後所有 tarball 自帶這版）

### Step 1 — 下載

```
https://github.com/alienid4/it-web-full/releases/download/v3.11.19.0/patch_v3.11.19.0.tar.gz
```

### Step 2 — 解壓 + 覆蓋（這次直接用單檔 cp, 繞過舊版 backup bug）

```bash
cd /tmp
curl -LO https://github.com/alienid4/it-web-full/releases/download/v3.11.19.0/patch_v3.11.19.0.tar.gz
tar xzf patch_v3.11.19.0.tar.gz

TS=$(date +%Y%m%d_%H%M)

# 覆蓋系統常駐的 patch_apply.sh (若有)
if [ -f /opt/inspection/scripts/patch_apply.sh ]; then
    sudo cp /opt/inspection/scripts/patch_apply.sh \
            /opt/inspection/scripts/patch_apply.sh.bak.${TS}
    sudo cp /tmp/v3.11.19.0/files/scripts/patch_apply.sh \
            /opt/inspection/scripts/patch_apply.sh
    sudo chown sysinfra:itagent /opt/inspection/scripts/patch_apply.sh
    sudo chmod 750 /opt/inspection/scripts/patch_apply.sh
fi

# 最重要: 從下一顆 patch (v3.11.20+) 開始, tarball 裡內建的 patch_apply.sh
# 自動就是新版 (我這邊 repo 已改源頭)。所以你下次 tar xzf 跑 ./vX/patch_apply.sh
# 就會用到改好的版本。
```

### Step 3 — 驗證

手動跑看看備份段：

```bash
sudo tar czf /tmp/debug_backup.tar.gz \
    --ignore-failed-read \
    --warning=no-file-ignored \
    --exclude=inspection/container \
    --exclude=inspection/logs \
    --exclude='*.sock' \
    --exclude='*.pid' \
    -C /opt inspection
echo "rc=$?"
ls -la /tmp/debug_backup.tar.gz
```

預期 `rc=0` + tar.gz 幾 MB 大小。
