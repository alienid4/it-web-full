# 2026-04-23 14:30 — v3.11.15.0 帳號盤點高權限 Phase 1

## 新功能（基礎：自動高權限標示）

### 1. Role 補抓 GID + 保留 root

`check_account_audit/tasks/linux.yml`：
- 新增 `gid` + `primary_group` 欄位（用 `getent group` 取組名）
- **UID=0 (root) 不再被 filter 掉**（原本 `uid < audit_min_uid(1000)` 全砍，現在 UID=0 例外保留）

### 2. 帳號盤點頁自動標示

`audit.html`：
- 表格加 **UID/GID** 欄位
- 表格加 **鎖定** 欄位（🔓 啟用 / 🔒 已鎖定）
- **UID=0 或 GID=0** 的帳號自動：
  - 紅底 row（`#FFEBEE`）
  - 帳號名旁 `🔴 高權限` badge
- 列總數旁顯示「其中 N 個高權限」
- filter 列加 **「🔴 只顯示高權限」** toggle

### 3. 保留上版功能

v3.11.14.0 的 /etc/passwd 檢視區塊保留不動。

## 套用（13 上）

### Step 1 — 下載 tarball

```
https://github.com/alienid4/it-web-full/releases/download/v3.11.15.0/patch_v3.11.15.0.tar.gz
```

### Step 2 — 解壓 + 套用

```bash
cd /tmp
curl -LO https://github.com/alienid4/it-web-full/releases/download/v3.11.15.0/patch_v3.11.15.0.tar.gz
tar xzf patch_v3.11.15.0.tar.gz
sudo ./v3.11.15.0/patch_apply.sh
```

### Step 2 備援（若 patch_apply.sh 仍卡 Step 1/5 備份）

**2 檔單檔 cp**：

```bash
TS=$(date +%Y%m%d_%H%M)

# role (ansible)
sudo cp /opt/inspection/ansible/roles/check_account_audit/tasks/linux.yml \
        /opt/inspection/ansible/roles/check_account_audit/tasks/linux.yml.bak.${TS}
sudo cp /tmp/v3.11.15.0/files/ansible/roles/check_account_audit/tasks/linux.yml \
        /opt/inspection/ansible/roles/check_account_audit/tasks/linux.yml
sudo chown sysinfra:itagent /opt/inspection/ansible/roles/check_account_audit/tasks/linux.yml

# audit.html (webapp)
sudo cp /opt/inspection/webapp/templates/audit.html \
        /opt/inspection/webapp/templates/audit.html.bak.${TS}
sudo cp /tmp/v3.11.15.0/files/webapp/templates/audit.html \
        /opt/inspection/webapp/templates/audit.html
sudo chown sysinfra:itagent /opt/inspection/webapp/templates/audit.html

# 重啟 Flask
sudo systemctl restart itagent-web
```

### Step 3 — 重跑 inspection 抓新 GID 資料

**關鍵**：role 改了，但舊 account_audit collection 沒 gid 欄位 → 現有資料在 UI 顯示 `UID/?`。要跑一次 run_inspection.sh 讓 ansible 重新採集：

```bash
sudo -u sysinfra /opt/inspection/run_inspection.sh
```

等約 2~5 分鐘。跑完後 account_audit collection 應該有：
- 每筆 `gid` 欄位
- 新增 root 這筆（UID=0）

### Step 4 — 驗證

打開 `http://10.92.198.13:5000/audit`：

1. 表格多了 **UID/GID** 和 **鎖定** 欄位
2. root 帳號出現，整列紅底 + `🔴 高權限` badge
3. 勾選「只顯示高權限」→ 只剩 UID=0 / GID=0 的
4. 上方統計顯示「共 N 個帳號，其中 X 個高權限」
