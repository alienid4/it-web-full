## 未解問題 (獨立議題, 不影響 v3.11.22.0)

❌ **sec9c2 (192.168.1.223) 缺 passwordless sudo** — onboarding 漏做
- 現象: `sudo: 需要密碼`
- 原因: `/etc/sudoers.d/ansible_svc` 只有 `ansible_svc NOPASSWD: ALL`, 缺 `sysinfra NOPASSWD: ALL`
- 對照: secclient1 兩條都有
- 影響: 不只深度檢查, 任何 ansible become:true 對 sec9c2 的操作都掛
- 修法 (sec9c2 上):
  ```
  echo 'sysinfra ALL=(ALL) NOPASSWD:ALL' | sudo tee -a /etc/sudoers.d/ansible_svc
  sudo visudo -c
  ```

## 221 環境踩坑

### 路徑是 /seclog/AI/inspection (不是 /opt/inspection)

221 的 inspection 家在 `/seclog/AI/inspection/`，`/opt/inspection/` 是空殼。
apply 腳本與 patch install.sh 都做了 **自動偵測** (優先 /opt, 後備 /seclog)。

### Git remote 不一致

本機 = `alienid4/it-web-full`，221 = `alienid4/CAS_IT`。
但 report.html 實際內容 md5 相同，所以本機改動可直接同步到 221。

### gunicorn 不是 systemd 而是 foreground + nohup

221 沒有 inspection systemd service。gunicorn 靠 sysinfra 手動啟動 + nohup 運行。
install.sh 做了優雅處理：優先 systemctl restart (依序試 itagent-web / inspection / inspection-web)，
後備 pkill + relaunch。

### chattr +a 合規 log 擋 rm

smit_menu 會 `chattr +a LinuxMenu_main_YYYYMMDD.log` (T0 合規需求)。
playbook 清理前要先 `chattr -R -ai`, 不然 `rm -rf /tmp/smit_menu_*` 會失敗。
