# 2026-04-24 13:50 — 014T 深度檢查失敗修法 (同 sec9c2 老帳)

**症狀**：對 SECSVR198-014T 跑深度檢查 → `fatal: Missing sudo password`
**根因**：014T 的 `/etc/sudoers.d/ansible_svc` 只有 `ansible_svc NOPASSWD`，**缺 `sysinfra NOPASSWD`**
**影響**：不只深度檢查，**任何對 014T 的 ansible become:true 操作都會掛**（TWGCB 掃描、帳號盤點、Linux 初始化…）

## 修法（3 選 1）

### A. 從 13 直接 ssh root 修（最快，推薦）

```bash
ssh root@<014T-ip> "
  grep -q 'sysinfra.*NOPASSWD' /etc/sudoers.d/ansible_svc 2>/dev/null \
    || echo 'sysinfra ALL=(ALL) NOPASSWD:ALL' >> /etc/sudoers.d/ansible_svc
  chmod 440 /etc/sudoers.d/ansible_svc
  visudo -c
  echo '---'
  echo 'verify:'
  sudo -u sysinfra sudo -n whoami
"
```

預期末段印：
```
verify:
root
```

### B. 透過 ansible 修（若 13 root 不能 ssh 014T）

先確認 014T 有允許 root SSH，或 ansible_svc 還能 become（因為它自己有 NOPASSWD）:

```bash
cd /opt/inspection/ansible
sudo -u sysinfra ansible -i inventory/hosts.yml SECSVR198-014T \
  --become-user ansible_svc --become \
  -m shell -a "echo 'sysinfra ALL=(ALL) NOPASSWD:ALL' >> /etc/sudoers.d/ansible_svc; chmod 440 /etc/sudoers.d/ansible_svc; visudo -c" \
  --vault-password-file /opt/inspection/.vault_pass
```

### C. 人工到 014T 主機修

登入 014T，edit `/etc/sudoers.d/ansible_svc`，加一行 `sysinfra ALL=(ALL) NOPASSWD:ALL`，存檔，`visudo -c`。

瀏覽器重新整理 → 點 014T 深度檢查 → 不用再改 code / patch，應該直接跑得通。

## 順便：可能的其他主機

建議批量檢查**所有受監控主機**是否缺 `sysinfra NOPASSWD`：

```bash
cd /opt/inspection/ansible
sudo -u sysinfra ansible -i inventory/hosts.yml all \
  -m shell -a "sudo -n whoami 2>&1 || echo SUDO_FAIL_on_$(hostname)" \
  --vault-password-file /opt/inspection/.vault_pass 2>&1 | tail -30
```

**任何回 `SUDO_FAIL_...` 的主機**都要套 A 的修法。
