# 2026-04-24 09:30 — 深度檢查 POST `/run` 回 HTML 診斷

**目前確認**：
- ✅ 安裝完整（version 3.11.22.0、blueprint 2 行都在、檔案都存在、Flask 運行中）
- ✅ GET `/api/deep-check/reports` 回 JSON 401
- ❌ POST `/api/deep-check/run` 回 HTML（造成 `Unexpected token '<'`）

**嫌疑**：CSRF middleware / Flask-WTF / 代理層 擋 POST。

## 判讀表

| [B] POST /run | [C] POST /linux-init/run | 結論 |
|---|---|---|
| **401 + CT=application/json** | 任意 | ✅ POST endpoint 本身 OK，問題在 browser 端（cookie / CSRF token 丟失）→ 看 [A]/[F] |
| **400 + CT=text/html** (含 "CSRF" 字樣) | 同樣 400 | ❌ **CSRF 擋所有 POST**，要找 CSRF exemption 機制 |
| **400 + CT=text/html** (無 CSRF 字樣) | 200 / 401 | ❌ 我的 /run 的 JSON body 驗證 bug — 回 bad request |
| **500 + CT=text/html** | 任意 | ❌ Flask 本身 crash，看 [D] |
| **404 + CT=text/html** | 401 | ❌ route 沒註冊 — 奇怪，跟 [2] 衝突，重看 blueprint 註冊 |
