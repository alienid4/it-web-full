## 你之前問的兩個問題答案

### Q1: 詳細資訊 / summary 放哪？

**固定位置**（套完 22.2 後）：
- `/opt/inspection/data/deep_check_reports/ts_<hostname>_<YYYYMMDD_HHMMSS>_summary.txt`
- `/opt/inspection/data/deep_check_reports/ts_<hostname>_<YYYYMMDD_HHMMSS>_detail.txt`

UI modal 會自動：
- **預覽** summary.txt 在黑底區塊
- **下載** summary + detail 兩個按鈕

管理員也可直接 `ls` 該目錄查歷史報告。

### Q2: 為什麼第一次成功但看不到 summary？

**推測**：第一次 playbook 跑到 fetch 步驟時，把 summary/detail 寫到 `/seclog/AI/inspection/data/deep_check_reports/`（playbook hardcode 路徑），但 API 去 `/opt/inspection/data/deep_check_reports/` 找 → 找不到 → modal 顯示完成但無報告按鈕。

第二次失敗純粹是 `/seclog/AI/inspection/data/` 沒寫權限觸發 file 模組錯誤。

**驗證**（可選，套 22.2 前先看）：
```bash
ls -la /seclog/AI/inspection/data/deep_check_reports/ 2>/dev/null
echo '---'
ls -la /opt/inspection/data/deep_check_reports/ 2>/dev/null
```

如果 /seclog 那邊有 `ts_*_summary.txt`，就證實我的猜測。22.2 install.sh 會自動把它搬到 /opt/inspection/ 保留歷史。
