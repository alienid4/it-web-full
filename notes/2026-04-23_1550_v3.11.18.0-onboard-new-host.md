# 2026-04-23 15:50 — v3.11.18.0 新主機一鍵 onboard 腳本

## 做什麼

新腳本 `scripts/onboard_new_host.sh`，接受 `<hostname>` 參數，5 步全自動：

1. **驗證主機已在 DB hosts collection**（未建會擋）
2. **generate_inventory.py 重建** `ansible/inventory/hosts.yml`
3. **ansible ping 驗證 SSH 通**（失敗會印除錯提示 + exit）
4. **run_inspection.sh 完整巡檢**（site.yml + packages + seed + CIO snapshot）
5. **TWGCB 掃描 + 啟用 nmon**（curl 觸發 /api/twgcb/import + `nmon_enabled=true` 5min 間隔）

完成後印 6 個驗證 URL（Dashboard/audit/twgcb/packages/perf）。

---

## 使用流程（以後加任何主機都用這套）

### A. UI 先加主機（一樣）

打開 `/admin#hosts` → 點「新增主機」→ 填 hostname/IP/ssh_user/ssh_port/ssh_key 存檔。

### B. 一條指令 onboard

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh <hostname>
```

例：

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh SECSVR198-012T
```

約 3~7 分鐘結束，會印進度（[1/5]...[5/5]）。

### C. 驗證

打開腳本結尾印的 6 個 URL。

## 套用（13 上裝這腳本 — 1 檔）

### Step 1 — 下載

```
https://github.com/alienid4/it-web-full/raw/main/AI/scripts/onboard_new_host.sh
```

scp 到 13 的 `/tmp/`。

### Step 2 — 放位置 + 執行權限

```bash
sudo cp /tmp/onboard_new_host.sh /opt/inspection/scripts/onboard_new_host.sh
sudo chown sysinfra:itagent /opt/inspection/scripts/onboard_new_host.sh
sudo chmod 750 /opt/inspection/scripts/onboard_new_host.sh
```

### Step 3 — 試跑（看 usage 說明）

```bash
sudo /opt/inspection/scripts/onboard_new_host.sh
```

應該印 Usage 說明。

### Step 4 — 實際用

現在你要加第 3 台時：

1. UI `/admin#hosts` 新增主機（填表存檔）
2. SSH 到 13，跑：
   ```bash
   sudo /opt/inspection/scripts/onboard_new_host.sh <新 hostname>
   ```
3. 等 5 分鐘看結果 → 6 個頁面都有資料

## 腳本特點

- ✅ 每步 log 存 `/tmp/onboard_*_$$.log`，失敗不用重翻終端
- ✅ Step 3 SSH 失敗會印「常見原因」清單（ssh_user/key/fw）
- ✅ Step 4.5 TWGCB 錯不 block（可事後 UI 觸發）
- ✅ Step 5 nmon interval 可用環境變數覆蓋：`NMON_INTERVAL=15 sudo ./onboard_new_host.sh <h>`
- ✅ 完成後自動抓機器 IP 產驗證 URL
