# 13 裝 v3.17.11.0 + 重 import CSV 補業務系統 → 拓撲長出來

## 目標

修今早 debug_topology.sh v2 跑出的根因 — **`dependency_systems` 是空的**, 4 台主機都不在任何 host_refs, 採集連線都採到了但 dep_rel 寫不進邊。

v3.17.11.0 把「業務系統歸屬」搬到 CSV (= 資產表 single source of truth), import 時自動同步 dependency_systems, 拓撲節點自己長出來。

---

## Step 1. 裝 patch

```bash
gh release download v3.17.11.0 -p '*.tar.gz' -O /tmp/p11.tar.gz
cd /tmp && tar -xzf p11.tar.gz
cd combo_v3.17.11.0_csv_business_system && sudo bash install.sh
```

預期看到結尾 `✅ v3.17.11.0 部署完成` + 5 項 smoke 全綠 (AST/helper/欄位/HTTP/log)。

---

## Step 2. 開 feature_flags.dependencies (debug v2 也報這個 FAIL)

進 `/superadmin` → 模組管理 → `dependencies` 開 ON。沒這步 admin 拓撲管理 tab 進不去。

---

## Step 3. 下載新 CSV 範本 + 填業務系統欄

進 `/admin` → 主機管理 → 「下載 CSV 範本」(或 XLSX), 確認有「業務系統」欄 (Excel 範本第 8 欄, 中文「業務系統」, 標備註「資產名稱(自動避險/SPEEDY 等)」)。

把現在 4 台 (011T/013T/014T/015T) 都填上歸屬:

```csv
hostname,ip,os,os_group,status,environment,group,business_system,custodian,custodian_ad,department,division
SECSVR198-011T,...,Rocky Linux,rhel,使用中,測試,,巡檢系統,...
SECSVR198-013T,...,Rocky Linux,rhel,使用中,測試,,巡檢系統,...
SECSVR198-014T,...,Rocky Linux,rhel,使用中,測試,,巡檢系統,...
SECSVR198-015T,...,Rocky Linux,rhel,使用中,測試,,巡檢系統,...
```

> 全填同一個「巡檢系統」最快, 4 台會綁進同一個 dependency_system, 拓撲圖看到 1 個節點 + 內部互連邊。
>
> 想分開可以填: 011T/013T = 「巡檢主控」, 014T/015T = 「巡檢採集」, 拓撲就會有 2 個節點分群。

---

## Step 4. 重 import CSV

進 `/admin` → 主機管理 → 「匯入 CSV」, 上傳剛填好的檔。

預期 admin 介面回應:
```
成功匯入 4 台主機
同步業務系統 +1/~0/移動 0   ← v3.17.11.0 新加, 之前沒這行
```

或回 JSON 含 `sync_systems: {systems_created:1, systems_updated:0, hosts_moved:0}`。

---

## Step 5. 確認 dependency_systems 真的有了

```bash
sudo mongosh inspection --quiet --eval '
db.dependency_systems.find({}, {_id:0, system_id:1, display_name:1, host_refs:1, "metadata.apid":1}).forEach(printjson)
'
```

預期看到:
```json
{
  "system_id": "巡檢系統",
  "display_name": "巡檢系統",
  "host_refs": ["SECSVR198-011T","SECSVR198-013T","SECSVR198-014T","SECSVR198-015T"]
}
```

---

## Step 6. 重採集 → 看拓撲

```
1. /dependencies → 應該看到 1 個節點「巡檢系統」(中文 label)
2. 點 toolbar「📡 採集」→ 等 1-3 分
3. 點「📊 狀態」→ 應該看到 status:success, edges_added > 0
4. 重整 /dependencies → 拓撲節點底下應該有內部連線邊
```

---

## Step 7. (驗收) 重跑 debug_topology.sh v2

```bash
gh release download debug-topology-v2 -p debug_topology.sh -O /tmp/debug_topology.sh
sudo bash /tmp/debug_topology.sh
```

預期 quiet mode 只剩一行:
```
✅ 全部關鍵檢查通過
  畫面還是空白 → 看 DevTools Console + Network tab
```

或更好:
```
✅ 全部關鍵檢查通過
```

之前 3 個 [FAIL] 應該全消:
- ✓ dep_sys 不再空 (Step 4 灌進去)
- ✓ feature_flags.dependencies = ON (Step 2 開了)
- ✓ host_refs 反查 4/4 全綁 (Step 4 同步)

---

## Step 8. 設 cron (對應 SOP Step 5b/c)

進 `/admin` → 拓撲管理 tab → ⏰ 採集排程 (cron):
- interval: **30 分**
- ☑ 上班時間限定 (9-18 週一~五)
- limit_hosts: 留空 (全部)

按「儲存」→ 確認 `sudo crontab -u sysinfra -l | grep MARK_DEP_COLLECT` 有寫入。
