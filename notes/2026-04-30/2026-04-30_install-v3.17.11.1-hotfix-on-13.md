# 13 裝 v3.17.11.1 hot-fix → 救 v3.17.11.0 起不來

## 現況

v3.17.11.0 部署後 service 卡 activating + auto-restart, ImportError: cannot import name 'get_hosts_col' from services.mongo_service。本機 mongo_service.py 也有無限遞迴 bug 一起修。

## 路徑 A. 直接裝 v3.17.11.1 (建議)

```bash
gh release download v3.17.11.1 -p '*.tar.gz' -O /tmp/p11_1.tar.gz
cd /tmp && tar -xzf p11_1.tar.gz
cd combo_v3.17.11.1_mongo_helper_hotfix && sudo bash install.sh
```

預期看到結尾:
```
✅  v3.17.11.1 hot-fix 部署完成

下一步 (接 v3.17.11.0 SOP Step 2):
  1. /superadmin → 模組管理 → 開 dependencies
  ...
```

跑完接 [v3.17.11.0 SOP Step 2-7](2026-04-30_install-v3.17.11.0-on-13.md) 繼續。

---

## 路徑 B. 萬一 v3.17.11.1 也壞 → 立即 rollback

```bash
ls -t /var/backups/inspection/v3.17.11.0_* | head -1   # 找 v3.17.11.0 的備份 (有 v3.17.10.3 的檔)
LATEST=$(ls -t /var/backups/inspection/v3.17.11.0_* | head -1)
sudo cp -r "$LATEST/webapp/"* /opt/inspection/webapp/
sudo systemctl restart itagent-web
```

回到 v3.17.10.3 (拓撲不會修但服務活著, dep_sys 仍 0)。

---

## 確認 service 真活著

```bash
sleep 5 && sudo systemctl status itagent-web --no-pager -l | head -10
curl -sS -o /dev/null -w "HTTP %{http_code}\n" --max-time 5 http://localhost:5000/login
```

預期 `Active: active (running)` + `HTTP 200`。

---

## 如果還有錯

```bash
sudo journalctl -u itagent-web --since "2 min ago" --no-pager | grep -A 20 -iE 'traceback|exception|error' | tail -40
```

把輸出貼回對話。
