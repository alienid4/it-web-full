# 2026-04-27 16:00 — 13 上 v3.13.0.1 兩個按鈕 fail 診斷

目的：admin#backups 頁的「立即備份」(程式備份/createBackup) 跟「立即 Dump」(DB Dump/createDbDump) 都 fail，需要實際錯誤訊息才能對症下藥。

**機器**：10.92.198.13 (RPM 裝 MongoDB, mode=local)
**版本**：3.13.0.1
**症狀**：
- 「立即備份」→ tar czf 'returned non-zero exit status 2'
- 「立即 Dump」→ ?（需要實際錯誤）

---

## 步驟 1：看 mongodump 環境

```bash
# 確認 mongodump 真的裝了
which mongodump
mongodump --version | head -3

# 確認 MongoDB 有沒有開 auth
sudo grep -E '^\s*authorization|^\s*security' /etc/mongod.conf 2>/dev/null
sudo systemctl status mongod | head -10
```

## 步驟 2：手動跑 mongodump（看真錯誤）

```bash
# 用 sysinfra (Flask 跑的 user) 試
sudo -u sysinfra mongodump --db inspection --archive --gzip > /tmp/test_dump.gz 2>/tmp/mongodump_err.log
echo "exit=$?"
ls -lh /tmp/test_dump.gz
cat /tmp/mongodump_err.log
```

預期：
- exit=0 + .gz 檔有大小 + err.log 空 → mongodump 自己沒問題，是 Flask 呼叫出錯
- exit≠0 + err.log 有錯誤 → MongoDB auth / 連線 / 權限問題

## 步驟 3：看 Flask 後端真錯誤

```bash
# 最近 100 行 + 抓所有錯誤關鍵字
sudo journalctl -u itagent-web -n 100 --no-pager > /tmp/flask_log.txt
grep -iE 'error|exception|traceback|tar|mongodump' /tmp/flask_log.txt | tail -40
```

或直接看完整 log：

```bash
sudo journalctl -u itagent-web -n 50 --no-pager
```

## 步驟 4：確認檔案權限

```bash
# Flask 跑的 user
ps -o user,group= -p $(pgrep -f gunicorn | head -1)

# 兩個 backup 目錄
ls -la /var/backups/inspection/
ls -la /var/backups/inspection/db/

# 程式備份來源 (createBackup 跑 tar czf -C /opt inspection/)
sudo ls -la /opt/inspection/.vault_pass        # 600 root 是不是讓 sysinfra 讀不到?
sudo find /opt/inspection -type f -not -readable 2>&1 | head -10
```

## 步驟 5：手動跑 createBackup 的 tar（看真錯誤）

```bash
sudo -u sysinfra tar czf /tmp/test_backup.tar.gz -C /opt inspection/ 2>/tmp/tar_err.log
echo "exit=$?"
cat /tmp/tar_err.log
ls -lh /tmp/test_backup.tar.gz 2>/dev/null
```

---

## 把結果貼回對話 (或開 GitHub issue)

優先序：
1. **步驟 3 的 log** — 最直接看到 Flask traceback
2. **步驟 5 的 tar_err.log** — 看 createBackup tar 為何 exit 2
3. **步驟 2 的 mongodump_err.log** — 看 DB Dump 為何 fail

不用每步都跑，**先跑步驟 3** 通常就找到原因。

## 預期可能根因 (給我參考)

| 症狀 | 根因 | 修法 |
|---|---|---|
| tar exit 2, 提到 .vault_pass | sysinfra 讀不到 root:root 600 | tar 加 `--ignore-failed-read` (v3.13.0.2) |
| tar exit 2, 提到 logs/ | 跑巡檢時 log 被寫 | tar 加 `--exclude='*.log'` 或 `--ignore-failed-read` |
| mongodump 'Authentication required' | MongoDB 開了 auth | mongodump 加 `--username/--password` 或關 auth |
| mongodump 'Failed: connection()' | port / auth source 對不上 | 加 `--port 27017 --authenticationDatabase admin` |
| Flask permission denied 寫 /var/backups/ | dir owner 不對 | chown 修 |
