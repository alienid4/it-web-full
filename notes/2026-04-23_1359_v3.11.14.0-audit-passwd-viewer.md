# 2026-04-23 13:59 — v3.11.14.0 帳號盤點加 /etc/passwd 原文檢視

## 新功能

帳號盤點頁 `/audit` 底下新增「**📄 /etc/passwd 原文檢視**」區塊：

- 主機下拉選一台
- 按「📄 查看」
- 即時透過 ansible `command -a "cat /etc/passwd"` 抓原文
- 以 dark-mode `<pre>` 顯示（等寬字體、可捲動、最高 420px）
- 上方 status 顯示「共 N 個帳號 + 載入時間」

## 檔案改動

| 檔 | 內容 |
|---|---|
| `webapp/routes/api_audit.py` | 新 endpoint `GET /api/audit/passwd?hostname=X`；regex 白名單防 shell injection；30 秒 timeout |
| `webapp/templates/audit.html` | 底部加 card 區塊 + `viewRemotePasswd()` JS + populate 下拉 |

## 設計選擇（為什麼這樣做）

- **即時抓 vs 預存**: 即時。永遠最新，資安用途不能信 cache。
- **單選 vs 多選**: 單選。看 passwd 通常一次聚焦一台。
- **ansible 不用 -b (sudo)**: `/etc/passwd` 全域可讀，無需 sudo 減少攻擊面。

## 常見錯誤

| 狀態 | 原因 | 解法 |
|---|---|---|
| `ansible failed` + stderr 含 "Permission denied" | sysinfra 沒 SSH key 到該主機 | 見 project_home_parity.md 的 SSH key 分發 SOP |
| `unexpected output` | ansible 版本差異 or 主機 stdout 格式異常 | 貼 stderr 給我看 raw |
| `timeout (>30s)` | 目標主機卡 or 網路慢 | 單獨 SSH 驗 `sudo -u sysinfra ansible <hostname> -m ping` |
